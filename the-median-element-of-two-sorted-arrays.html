This is a LeetCode problemL1.

Problem Statement

There are 2 sorted arrays A and B of size m and n respectively. Write an algorithm to find the median of the array obtained after merging the above 2 arrays.

Clarifying questions

  1. Q: How is the median calculated when there are an even amount of elements?
    A: It is the average of the two middle elements.

Background Knowledge

The Median

In probability theory and statistics, a median is described as the number separating the higher half of a sample, a population, or a probability distribution, from the lower half. The median of a finite list of numbers can be found by arranging all the numbers from lowest value to highest value and picking the middle one.

For getting the median of input array \(\{12, 11, 15, 10, 20\}\), first sort the array. The sorted input array is \(\{10, 11, 12, 15, 20\}\). The Median is the middle element of the sorted array which is \(12\).

There are different conventions to take median of an array with even number of elements, one can take the mean of the two middle values, or first middle value, or second middle value.

Solututions

The following definitions apply to all the solutions:

  • \(M\): The median, \(M := \dfrac{m+n-1}{2}\)

Merge Arrays And Iterate

  • Time: \(O(m+n)\)
  • Space: \(O(m+n)\)

The idea is to merge the two sorted arrays into one array, and then iterate until the median element is found. This is very simple to impement, but very inneficient

Use Merge Routine From Merge SortL2

  • Time: \(O(m+n)\)
  • Space: \(O(1)\)

Space efficiency is improved with this method.

int mergeIterate(int *A, int *B, int m, int n) {
    static int Ai=0, Bi=0;
    int result = -1;

    if (Ai < m && (Bi >= n || A[Ai] <= B[Bi])) {
        result = A[Ai];
        Ai++;
    } else if (Bi < n && (Ai >= m || B[Bi] < A[Ai])) {
        result = B[Bi];
        Bi++;
    }

    return result;
}

double findMedianSortedArrays(int A[], int m, int B[], int n) {
    int M = (m+n)/2;
    double result;

    for (int i=0; i < M; i++)
        result = mergeIterate(A, B, m, n);

    result += (!((m+n)%2)) ? result : mergeIterate(A, B, m, n);
    return result/2.0;
}

Binary Search (Optimal Solution)

  • Time: \(O(\log\cdot m + \log\cdot n)\)
  • Space: \(O(1)\)

The optimal solution uses binary searchL3

Explanation

An index \(i\) in array \(A\) can be tested to determine if it is the median in the merged array in \(O(1)\) time:

  • If index \(i\) in \(A\) is selected, then it can only be the median element in the entire array if \(A[i]\) is in the \(M\)'th position in the entire merged array.
  • By definition, \(A[i]\) is greater than or equal to \(i\) elements in \(A\). It must also be greater than or equal to \(M-i-1\) elements in \(B\)1.
  • Set \(j = M-i-1\), and test if \(A[i] \geq B[j]\). This is a predicate function that can be used in binary search.
  • Find the first true result, and if \(B[j] \leq A[i] \leq B[j+1]\), it is the median of the entire merged array. If not, then the median element lies in array \(B\) and so the whole procedure must be repeated for \(B\).

In the code below, the predicate function is medianTest. Here is a line-by-line explanation of it:

  1. b1 is the j'th element in B and if j is in the bounds of B, set it to B[j]. The ternary operator will set it to INT_MIN if not in bounds.
  2. b2 is the j+1'th element in B, and if in bounds, then it is set to B[j+1]. The ternary operator will initialize it to INT_MAX if not in bounds.
  3. The predicate tested for in line 4 only takes into account b1. If j is greater than the bounds of B, then b1's initialized value must be changed to INT_MAX. If on the other hand j is less than 0, it can just stay at the initialized value of INT_MIN
  4. The predicate check. The aim of this binary search will be to find the first A[i] that is greater than the corresponding b1. If j is less than 0, then this is automatically true (so set b1 to INT_MIN). If j is greater than B's bounds, then this is automatically false (so set b1 to INT_MAX).

bool medianTest(int *A, int *B, int n, int i, int j, int &b1, int &b2) {
    b1 = (j >= 0 && j < n) ? B[j] : INT_MIN;
    b2 = (j+1 >= 0 && j+1 < n) ? B[j+1] : INT_MAX;
    if (j >= n) b1 = INT_MAX;
    return (A[i] >= b1);
}

bool findMedian(int *A, int m, int *B, int n, double &median) {
    if (!m) return false; //base case
    int M = (m+n-1)/2, lo = 0, hi = m-1, b1, b2;
    while (lo < hi) {
        int i = lo + (hi-lo)/2, j = M-i-1;
        if (medianTest(A, B, n, i, j, b1, b2))
            hi = i;
        else
            lo = i+1;
    }

    if (medianTest(A, B, n, lo, M-lo-1, b1, b2) && b1 <= A[lo] && A[lo] <= b2) {
        median = A[lo];
        if (!((m+n)%2)) {
            median = (median+((lo+1 < m) ? min(b2, A[lo+1]) : b2))/2.0;
        }
        return true;
    }
    return false;
}

double findMedianSortedArrays(int A[], int m, int B[], int n) {
    double result;
    (findMedian(A,m,B,n,result)||findMedian(B,n,A,m,result));

    return result;
}
