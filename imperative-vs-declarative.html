Imperative Vs Declarative

  • Published: 05 Apr 2015

Definitions

Imperative

Imperative programming: tells the “machine” how to do something, and as a result what you want to happen will happen.

Imperative programming is a paradigm that describes computation in terms of statements that change a program state. In much the same way that the imperative moodL1 in natural languages1 expresses commands to take action, imperative programs define sequences of commands for the computer to perform. In general, Imperative programming is focused on describing how a program operates.

Procedural Programming

A procedural paradigm means a program is built from one or more procedures (also known as subroutines or functions). The idea is that state changes are localized to procedures or restricted to explicit arguments and returns from procedures (this is known as [structured programming](http://en.wikipedia.org/wiki/Structured_programming. Structured programming is normally promoted to improve the maintainability and overall quality of imperative programs.

Object Oriented ProgrammingL2 is just an extension of the structured pogramming paradigm.

Disadvantages Of Imperative Programming

Declarative

Declarative programming: tells the “machine” what you would like to happen, and lets the machine figure out how to do it.

Declarative programs are often described as a program that is not imperative, although a number of common definitions do exist:

Declarative Programming Definitions

  1. A program that describes what computation should be performed and not how to compute it
  2. Any programming language that lacks side effectsL3 (or more specifically, is referentially transparentL4)
  3. A language with a clear correspondence to mathematical logic.
